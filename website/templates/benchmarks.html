When to Use What

ScenarioRecommendedWhy
Real-time / streaming Cleora Fastest for re-embedding. Constant memory. Incremental updates.
Quick baseline RandNE Sub-millisecond. Good for initial exploration.
Best accuracy NetMF / DeepWalk Consistently top performers across datasets.
Small graph, perfect results GraRep Multi-scale capture. 100% on Karate Club.
Homophily tuning Node2Vec p/q parameters control BFS vs DFS exploration.
Directed graphs HOPE Preserves asymmetric relationships.
Memory constrained Cleora / RandNE Minimal memory footprint (< 0.1 MB for small graphs).

Methodology

  • Task: Node classification with 80/20 train-test split
  • Classifier: Nearest Centroid (no hyperparameters)
  • Embedding dimension: 128 for all algorithms
  • Metrics: Accuracy, Macro F1, wall-clock time, peak memory delta
  • Hardware: Single CPU core, no GPU
  • Datasets: Community-structured synthetic graphs matching real-world properties
  • Walk-based params: num_walks=20, walk_length=40 (small graphs); num_walks=3, walk_length=15 (large graphs)
Reproduce: Run python3 benchmark_report.py from the pycleora repository to regenerate all results.
