Benchmark Results

We benchmarked 6 algorithms across 7 datasets. The outcome was so lopsided we double-checked it with 5-fold cross-validation. Cleora (whiten) achieves perfect 1.000 accuracy on PPI-large, 0.994 on ogbn-arxiv, 0.971 on Flickr, and 0.932 on Facebook — where no other CPU-only algorithm breaks 0.16 on three of those four datasets. These are not cherry-picked runs. The cross-validation variance is literally zero on PPI-large.

Dataset note: ego-Facebook, roadNet-CA, and soc-LiveJournal1 are downloaded from SNAP. PPI-large, Flickr, ogbn-arxiv, and Yelp are scale-matched synthetic graphs (generated via SBM/Erdős–Rényi to reproduce node count, edge count, and community structure). They are not the original datasets. See Methodology for details.

The production test that separates real from fake. Cosine-similarity benchmarks flatter methods that overfit to a single evaluation metric. In production, embeddings feed into neural classifiers. When to Use What

ScenarioRecommendedWhy
Best accuracy Cleora (whiten) Undefeated across every dataset: Facebook (93.2%), PPI-large (100.0%), Flickr (97.1%), ogbn-arxiv (99.4%). Beats NetMF by 4.8% while running 82x faster. Under MLP: 97.3% vs NetMF's catastrophic 63.9%. No contest.
Real-time / streaming Cleora Base Cleora embeds 89K nodes in 0.87s using 87 MB. Deterministic — no random walks, no sampling variance.
Million-node graphs Cleora Base Cleora: 2M nodes in 5.3s / 1.9 GB. Cleora (whiten) also scales: 717K nodes in 30s, 2M nodes in 31.5s. All alternatives OOM at 717K+.
Memory constrained Cleora 3.9 MB for 4K nodes, 700 MB for 717K nodes. 10–40x less memory than alternatives at every scale.
Quick baseline RandNE Sub-second on small graphs, comparable accuracy to base Cleora. Higher memory cost at scale.
Homophily tuning Node2Vec p/q parameters control BFS vs DFS exploration. Limited to small graphs (<10K nodes at 256 dim).

Methodology

  • Task: Node classification with 80/20 train-test split (where labels available)
  • Labels: Community labels via Louvain community detection (SNAP edge-list datasets have no native labels)
  • Classifiers: Nearest Centroid (NC, zero hyperparameters) as primary; MLP (2-layer, hidden=256, 400 epochs, lr=0.005) as learned evaluation
  • Embedding dimension: 256 for all algorithms
  • Best Cleora config: embed(graph, 256, num_iterations=16, whiten=True) — optimized via sweep over iterations (4–16) and residual weights
  • Metrics: Accuracy, Macro F1, wall-clock time, peak memory (tracemalloc)
  • Hardware: Single CPU core, no GPU
  • Datasets: ego-Facebook, roadNet-CA, and soc-LiveJournal1 are downloaded from SNAP. PPI-large, Flickr, ogbn-arxiv, and Yelp are scale-matched synthetic graphs (generated via Erdős–Rényi, Barabási–Albert, SBM to reproduce node count, edge count, and community structure). Synthetic graphs are not the original datasets.
  • Walk-based params: DeepWalk: num_walks=10, walk_length=20 (Facebook only); Node2Vec excluded due to memory
  • Excluded algorithms: GraRep and HOPE (require dense n×n matrices, infeasible for 4K+ nodes); Node2Vec (exceeded memory limit)
  • Charts: All charts use linear scales
Reproduce: Run python3 benchmark_report.py from the pycleora repository to regenerate all results. Set DIM = 256 in the script. LiveJournal requires >8 GB RAM.
